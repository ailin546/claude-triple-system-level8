# 12 维场景探索流程

> 改编自 uditgoenka/autoresearch scenario 子命令，复用 test-pressure-*.md 维度模板和 fault-scenarios.js 扫描模式。

## 与现有机制的集成

### test-pressure-*.md（维度模板参考）

已有的 test-pressure 文件提供了场景编写的范式：
- test-pressure-1: 紧急生产修复情景
- test-pressure-2: 复杂集成情景
- test-pressure-3: 性能退化情景

autoloop:scenario 将这种手工场景编写自动化为 12 维系统性探索。

### fault-scenarios.js（扫描模式参考）

已有的 fault-scenarios hook 扫描 5 种模式。autoloop:scenario 扩展到 12 个维度。

## 12 维度详解

### 维度 1: NULL/EMPTY

```
目标：发现空值处理缺陷
检查项：
- 函数参数传 null/undefined
- 空字符串作为输入
- 空数组/空对象
- 可选参数全部省略
- 数据库返回 null 的字段
```

### 维度 2: BOUNDARY

```
目标：发现边界条件缺陷
检查项：
- 整数最大/最小值（MAX_SAFE_INTEGER, -1, 0）
- 字符串长度极值（空, 1字符, 超长）
- 数组大小极值（0, 1, 10000+）
- 分页参数（page=0, page=-1, limit=0, limit=999999）
- 日期范围（起止相同, 起止反转, 跨年/跨月）
```

### 维度 3: TYPE

```
目标：发现类型安全缺陷
检查项：
- 字符串传入数值字段
- 布尔值的 truthy/falsy 边界
- 浮点精度问题
- BigInt 与 Number 交互
- JSON 解析类型推断错误
```

### 维度 4: CONCURRENCY

```
目标：发现并发安全缺陷
检查项：
- 竞态条件（读-改-写不原子）
- 死锁（循环等待资源）
- 重入（函数被并发调用多次）
- 并发写入同一资源
- Promise.all 部分失败处理
```

### 维度 5: NETWORK

```
目标：发现网络韧性缺陷
检查项：
- 请求超时处理
- 连接断开后重连
- 重试策略（指数退避、最大重试）
- 部分响应处理
- DNS 解析失败
- CDN/代理层故障
```

### 维度 6: PERMISSION

```
目标：发现权限控制缺陷
检查项：
- 未授权访问
- 水平权限提升（A 用户访问 B 用户数据）
- 垂直权限提升（普通用户执行管理操作）
- 令牌过期处理
- 角色冲突（同时是管理员和被管理者）
```

### 维度 7: STATE

```
目标：发现状态管理缺陷
检查项：
- 无效状态转换（跳过中间状态）
- 脏数据（部分更新后崩溃）
- 缓存失效（stale cache）
- 幂等性（重复操作产生不同结果）
- 状态回滚不完整
```

### 维度 8: INJECTION

```
目标：发现注入漏洞
检查项：
- SQL 注入（' OR 1=1 --）
- XSS（<script>alert(1)</script>）
- 命令注入（; rm -rf /）
- 路径遍历（../../etc/passwd）
- LDAP 注入、NoSQL 注入
```

### 维度 9: RESOURCE

```
目标：发现资源耗尽缺陷
检查项：
- 内存泄漏（未释放的事件监听器）
- 磁盘空间不足
- 文件句柄耗尽
- 连接池用尽
- CPU 密集操作阻塞事件循环
```

### 维度 10: TIME

```
目标：发现时间相关缺陷
检查项：
- 时区处理（UTC vs 本地时间）
- 夏令时切换
- 闰年/闰秒
- 时钟偏移（分布式系统）
- TTL/过期时间计算
- 定时器精度（setInterval 漂移）
```

### 维度 11: ENCODING

```
目标：发现编码处理缺陷
检查项：
- Unicode 字符（emoji, CJK, RTL）
- 多字节字符截断
- BOM 标记处理
- 编码不匹配（UTF-8 vs Latin-1）
- URL 编码/解码
- Base64 填充
```

### 维度 12: DEPENDENCY

```
目标：发现外部依赖缺陷
检查项：
- 外部服务不可用时的降级策略
- API 版本不兼容
- 库升级的 breaking changes
- 数据库连接中断
- 第三方 SDK 行为变更
```
